# RanFH_coco

A YOLO-format object detection dataset for **frequency-hopping (FH)** signal detection in
low-SNR (signal-to-noise ratio) images, together with a conversion tool that produces
COCO-format annotations for evaluation.

The dataset is organized by SNR level (e.g. `SNR_-12`, `SNR_0`, `SNR_20`), which makes it
possible to evaluate detection performance across different noise conditions.

## Directory Structure

```
RanFH_coco/
├── classes.txt                 # class names (one per line), currently: FH
├── GenDataset_coco.py          # YOLO -> COCO JSON converter (Chinese comments)
├── GenDataset_coco_en.py       # English-commented version of the converter
├── images/
│   ├── train/                  # training images, named SNR_<db>_<idx>.png
│   └── val/                    # validation images, same naming scheme
└── labels/
    ├── train/                  # YOLO txt labels, one file per training image
    └── val/                    # YOLO txt labels, one file per validation image
```

Each image `SNR_<db>_<idx>.png` has a corresponding label file
`labels/<split>/SNR_<db>_<idx>.txt`.

## Annotation Format

### Classes

Single class: `FH` (frequency-hopping signal). `classes.txt` contains one line:

```
FH
```

### YOLO labels (`labels/<split>/*.txt`)

Each line is a normalized bounding box in YOLO format:

```
<class_id> <x_center> <y_center> <width> <height>
```

- `class_id`: integer, `0`-based (only `0` for the single `FH` class)
- all coordinates are relative to image width/height (`0.0`–`1.0`)

Example (`labels/train/SNR_-10_111.txt`):

```
0 0.0136 0.1993 0.0273 0.0111
0 0.1058 0.3882 0.0758 0.0111
...
```

### COCO annotations (`annotations/instances_val.json`)

Generated by `GenDataset_coco.py` / `GenDataset_coco_en.py`. The converter reads the YOLO
labels and writes a COCO-style dictionary with the standard `images`, `categories`, and
`annotations` fields. Bounding boxes are stored as absolute pixel `[x, y, w, h]`
(top-left corner). 

## Using the COCO Converter

The converter `GenDataset_coco.py` (or `GenDataset_coco_en.py`) turns YOLO labels into a
COCO JSON file used for mAP evaluation (e.g. with Ultralytics).

```bash
cd RanFH_coco
python GenDataset_coco_en.py
```

By default it converts `images/val` + `labels/val` and writes
`annotations/instances_val.json`.

To convert other splits, edit the paths in the `__main__` block:

```python
img_paths  = glob.glob(current_dir + '/val/*.png')   # image paths
label_path = current_dir + '/labels/val/'            # YOLO label directory
save_path  = current_dir + '/annotations'            # output directory
```

> Note: `save_json()` writes to a fixed file name `instances_val.json` and reads
> `save_path` from the global `__main__` scope. Adjust the function if you need a
> different output name or want to convert the training split.
